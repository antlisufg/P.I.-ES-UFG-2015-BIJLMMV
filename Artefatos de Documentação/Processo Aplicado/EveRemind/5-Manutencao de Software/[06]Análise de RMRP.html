[06]RMRP - AIM003

Tipo da Modificação

Corretiva

Análise de Impacto

A mudança afetará o código fonte, pois só há necessidade de alterar a lógica de verificação permitida para a hora da atividade. Afetará também a funcionalidade de enviar e-mails.

Impacto aos Usuários

Como a aplicação não foi liberada para usuários durante a descoberta da anomalia, não há impacto para os usuários.

Implicações/Limitações

Como implicação da mudança a aplicação terá um maior nível de confiança, pois não deixará que um usuário cadastre uma nova atividade para o passado. Não há limitações.

Estimativa de Esforço

Para realizar a mudança será necessário: Recurso Humano: 1 integrante da equipe de desenvolvimento Tempo: 1 hora de trabalho

Estimativa de Benefícios

A aplicação terá um maior nível de confiança.

Opções de Implementação

1- Implementar uma verificação no código-fonte para impedir que aplicação aceite um horário anterior ao do atual na data atual. Quando o usuário digitar um horário anterior ao atual, avisá-lo sobre e orientá-lo sobre os horários permitidos pela aplicação.

Plano de Prevenção e Mitigação dos Riscos

Não há riscos por se tratar de uma mudança simples.
